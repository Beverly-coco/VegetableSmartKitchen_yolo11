# “菜先知”智慧厨房系统 (前端部分)

## 1. 项目简介

“菜先知”是一款基于 `uni-app` 和 `Vue.js` 开发的跨平台智能应用，旨在为用户提供高效、便捷的蔬菜识别服务。用户通过手机拍照或上传图片，即可快速获取蔬菜的名称、置信度、营养成分及相关小知识。项目前端界面设计简洁、交互友好，后端集成深度学习模型，实现了从图像采集到智能分析的全流程体验。

**求职亮点**: 该项目完整地展示了作为一名前端开发者，独立负责一个完整小程序或 H5 应用的设计、开发、调试和部署全过程的能力。它不仅涵盖了前端开发的核心技能，如组件化、状态管理和 API 交互，还体现了对用户体验和性能优化的深入思考。

---

## 2. 技术栈

- **核心框架**: `Vue.js`
- **跨平台框架**: `uni-app` (一套代码，多端发布，包括 iOS、Android、H5 及各平台小程序)
- **UI**: 原生 `CSS` + `Flexbox` 布局，实现了高度定制化的美观界面
- **HTTP 通信**: 基于 `Promise` 封装的 `uni.request`，实现了模块化的 API 请求管理
- **状态管理**: `Vue` 组件的 `data` 属性和父子组件通信，用于管理页面和组件状态
- **路由管理**: `uni-app` 内置的页面路由管理 (`pages.json`)

## 4. 项目亮点与技术难点

### a. 跨平台开发与组件化实践

- **技术实现**: 基于 `uni-app` 和 `Vue.js`，实现了“一次开发，多端部署”的目标。项目遵循组件化开发的思想，将 UI 元素（如按钮、卡片）和业务逻辑（如图片上传、结果展示）封装成可复用的 `Vue` 组件。例如，在 `index.vue` 中，识别结果的展示被设计成一个循环渲染的卡片列表，每个卡片都是一个独立的展示单元。
- **求职价值**: 体现了对现代前端工程化和模块化开发的深刻理解，具备快速构建高质量跨平台应用的能力。

### b. 异步数据流与状态管理

- **技术实现**: 项目中广泛使用 `async/await` 来处理异步 API 请求，代码逻辑清晰易读。在 `utils/api.js` 中，对 `uni.request` 进行了二次封装，统一处理了 `Token` 注入、错误捕获和 `Toast` 提示，极大地提高了代码的健壮性和可维护性。例如，在 `index.vue` 的 `uploadImage` 方法中，通过 `isLoading` 状态来控制按钮的禁用和加载动画的显示，优化了用户等待期间的交互体验。
- **求职价值**: 证明了开发者能够熟练处理复杂的前端异步逻辑，并具备良好的状态管理和用户体验优化能力。

### c. 前后端分离与 API 设计

- **技术实现**: 前端通过 `utils/api.js` 模块与后端进行清晰的分离。该文件统一管理了所有 API 接口，包括用户认证、图片上传、模型训练等。通过 `Promise` 封装，使得业务代码可以链式调用，逻辑更加清晰。例如，`login` 函数封装了登录请求的全部细节，业务组件只需调用 `login({ username, password })` 即可。
- **求职价值**: 展现了在前后端分离架构下的协作能力，以及设计和维护模块化、可扩展 API 层的实践经验。

### d. 用户认证与路由拦截

- **技术实现**: 在 `App.vue` 的 `onLaunch` 生命周期中，通过检查本地缓存的 `token`，实现了全局的路由拦截。如果用户未登录，则自动跳转到登录页，确保了应用核心功能的安全性。在 `utils/api.js` 中，封装的请求函数会自动在请求头中携带 `Token`，并在 `401` 状态码时将用户重定向到登录页。
- **求职价值**: 体现了对前端安全和用户认证流程的深刻理解，能够设计和实现安全、可靠的客户端认证机制。

## 5. 如何运行

1.  **克隆项目**

    ```bash
    git clone <your-repo-url>
    cd caixianzhi_frontend
    ```

2.  **安装依赖**

    本项目没有使用 `npm` 依赖，`uni-app` 的核心库已内置。

3.  **在 HBuilderX 中运行**

    - 打开 HBuilderX，将 `caixianzhi_frontend` 文件夹拖入项目管理器。
    - 配置 `utils/api.js` 中的后端 `BASE_URL` 地址。
    - 点击菜单栏的“运行”，选择“运行到浏览器”或“运行到小程序模拟器”即可。

4.  **打包发布**

    - 点击菜单栏的“发行”，选择目标平台（如 H5、微信小程序等），按提示完成打包。
